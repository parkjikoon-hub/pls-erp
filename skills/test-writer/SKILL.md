## 기능 설명
pytest + pytest-asyncio 기반 ERP 테스트 코드를 표준 패턴으로 생성합니다.

## 테스트 카테고리

1. **단위 테스트**: 비즈니스 로직, 계산 엔진 (예: 급여 계산)
2. **API 테스트**: 엔드포인트 정상/에러/권한 케이스
3. **통합 테스트**: 모듈 간 연계 흐름 (예: 수주→생산→재고)
4. **보안 테스트**: RBAC 권한 차단, Audit Log 기록 검증

## 필수 테스트 케이스 패턴

```python
# 각 기능별 3가지 케이스 필수
def test_{기능}_정상케이스():    # Happy Path
def test_{기능}_에러케이스():    # Error Case
def test_{기능}_권한케이스():    # RBAC Check
```

## 사용 예제

예제 1: API 테스트 생성
- 입력: "M4 세금계산서 API 테스트 만들어줘"
- 출력: test_tax_invoice.py (생성/발행/국세청전송/권한 케이스)

예제 2: 계산 로직 단위 테스트
- 입력: "급여 계산 엔진 테스트 만들어줘"
- 출력: 비과세 최적화/법정한도/실수령액 검증 테스트

예제 3: 전체 테스트 실행
- 입력: "전체 테스트 실행하고 결과 보고해줘"
- 출력: pytest 실행 + 커버리지 리포트 요약
